Bridge plugins

One contract, different engines.

Start with Codex today. The same pattern leaves room for Gemini and Cursor without changing how you brief, constrain, or collect delegated work.

What stays consistent

Each bridge writes an auditable Markdown spec, points the external agent at that file, and keeps project standards grounded in this repo.

Available
codex

Delegate image generation, hard-reasoning offload, browser checks, and raw Codex CLI execution when Codex is the stronger tool for the task.

The handoff

Five sections, no surprise context loss.

Claude Code resolves ambiguity before the external agent runs. The final spec is explicit, reviewable, and stored on disk.

1

What to do

The role and task, expanded from your request.

2

How to do

Numbered steps or open-ended delegation.

3

Constraints

Links to repo standards like code style, design, and security.

4

Output

Expected format, artifact path, and success criteria.

5

Clarifications

Open questions resolved before delegation starts.

Install

Three commands in Claude Code.

1. Add the marketplace 
/plugin marketplace add https://github.com/raiaman15/carefully-crafted-plugins
2. Install the bridge 
/plugin install codex@carefully-crafted-plugins
3. Scaffold standards 
/codex:setup

What setup creates Starter Markdown files under docs/carefully-crafted-plugins/{constraints,output-formats}/ so each project can encode its own standards once and reuse them across every bridge.
